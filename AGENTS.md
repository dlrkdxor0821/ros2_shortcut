# AGENTS.md

이 저장소에서 AI(코딩 에이전트)에게 작업을 시킬 때, 아래 규칙을 우선으로 따릅니다.

## 프로젝트 목적
- ROS 2 **Jazzy** 환경에서 토픽/서비스를 “웹에서 보기” 위한 구성을 정리한다.
- 특히 **Foxglove Studio + `foxglove_bridge`** 조합으로 **이미지/3D(TF, PointCloud2, RobotModel)** 시각화를 쉽게 한다.

## 산출물(우선순위)
- **문서 우선**: README와 `docs/`에 개념/네트워크/패널 설정을 명확히 정리한다.
- **재현 가능성**: 특정 환경에 종속된 값은 “예시”로 표기하고, 사용자가 바꿔야 하는 지점을 분명히 한다.
- **보안 인지**: LAN까지는 기본 지원, “인터넷 원격 공개”는 보안 항목(인증/터널링/방화벽)을 별도로 다룬다.

## 문서 스타일 가이드
- Markdown은 GitHub에서 바로 읽히게 구성한다.
- 과도한 장문보다 **체크리스트/표/짧은 섹션**을 선호한다.
- 코드/파일 경로/토픽명/명령은 백틱(`)을 사용한다.
- 다이어그램이 도움이 되면 `mermaid`를 사용한다(복잡한 스타일링은 피함).

## 기술 가정/기본값
- ROS 2 배포판: **Jazzy**
- 브리지: **`foxglove_bridge`**
- 뷰어: **Foxglove Studio**
- 네트워크는 기본적으로 **같은 PC 또는 같은 LAN**을 우선 타겟으로 한다.
- `ROS_DOMAIN_ID`는 “웹 제공”이 아니라 **발견/분리 목적**임을 문서에서 명확히 한다.

## 파일 구조 원칙
- 사용자 가이드 문서는 `docs/`에 둔다.
- 설정/런치 예시는 `ros/` 아래(추후 추가 가능).
- 스크립트는 `scripts/` 아래(추후 추가 가능).

## 작업 시 주의사항
- 새로운 파일/폴더를 추가할 땐 README의 “추천 폴더 구조”와 일치하게 유지한다.
- 특정 패키지 설치 커맨드는 배포판/환경에 따라 다를 수 있으니, **확정이 어려우면 단정하지 말고 확인 방법**을 같이 적는다.
